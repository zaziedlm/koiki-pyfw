# JP1 Return Code Mapping

このドキュメントは、バッチアプリの終了コードを JP1/AJS の判定条件へマッピングするための標準を示します。

## 1. 標準終了コード

| Code | 種別 | JP1判定 | 意味 | 代表アクション |
|---:|---|---|---|---|
| 0  | Normal | 正常終了 | 全処理成功 | 後続ジョブへ進む |
| 10 | Warning | 警告終了 | 一部スキップや軽微不整合（業務継続可） | 通知のみ、原則後続継続 |
| 20 | Business Error | 異常終了(業務) | 入力不備・業務条件未達など | 手順書に従い業務確認後リラン |
| 30 | System Error | 異常終了(システム) | DB/ネットワーク/アプリ障害 | 基盤調査・復旧後リラン |

> 補足: 実アプリが 0/10/20 以外を返した場合、`run-job.sh` 側で 30 に正規化します。

## 2. JP1 設定指針

- 正常判定: `0`（必要に応じて `0,10` を正常継続扱いに設定）
- 警告判定: `10`（通知を発火）
- 異常判定: `20,30`
- 監視連携: 30 は運用一次連絡先に即時通知

## 3. アプリ実装指針

1. 例外分類を「業務エラー」と「システムエラー」に分離する。
2. ジョブ終了時に最終判定コードを必ずログ出力する。
3. リラン可否（同日再実行可/不可）をジョブ単位で定義する。
4. 警告コード(10)を返す条件は運用と合意して固定する。

## 4. 代表ログ例

```text
[INFO ] START 2026-05-19 01:00:00 job=customer-daily-sync env=prd requestId=JP1-20260519010000 bizDate=20260519
[ERROR] Business validation failed: missing billing cycle
[INFO ] END   2026-05-19 01:01:12 job=customer-daily-sync env=prd requestId=JP1-20260519010000 appRc=20
[INFO ] JP1_RC=20 (from APP_RC=20) log=/var/log/koiki-batch/customer-daily-sync_20260519_010000.log
```

## 5. 変更管理

- 終了コード体系の変更は、必ず下記を同時更新すること。
  - `ops/jp1/scripts/run-job.sh`
  - JP1 ジョブネット定義
  - 運用手順書（一次対応・エスカレーション）
