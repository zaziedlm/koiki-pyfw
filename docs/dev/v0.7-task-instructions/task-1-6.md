## 確認観点

- root、`libkoiki`、`app` の責務が説明できるか
- `setup.py` 廃止可否が判断できているか
- runtime/dev/test の境界が見えているか

2. 未解決事項を一覧化する
3. 未解決事項が Stage 2 着手を妨げるかを判定する
4. 妨げるものがあれば Stage 1 に差し戻す
5. 問題なければ Stage 2 着手可と記録する

## 検証

- `uv` 化の前提に致命的な未解決事項がない
