# KOIKI-FW v0.7 existing docs cleanup follow-up plan

## 目的

Stage 2 の `uv` 実移行と follow-up により、依存管理、Docker、CI、security audit、主要 runbook の標準経路は `uv` に切り替わった。

次に、実行時 warning や OSS 依存脆弱性の修正へ進む前に、既存 Markdown / docs / agent guidance / tool history に残る古い前提を整理する。

この文書は、不要または齟齬のある文書・履歴・agent/tool 由来ファイルを段階的に整理するための follow-up 計画である。

## 現在の基準

現行の標準経路:

- 依存同期: `uv sync --locked`
- 実行: `uv run --locked ...`
- lockfile 正本: `uv.lock`
- reusable framework 正本: `components/libkoiki`
- reference app 正本: `components/koiki_ref_app`
- application ASGI 正本: `koiki_ref_app.asgi:app`
- root `app/`: 互換 wrapper

現行文書として残すべきではない前提:

- Poetry を標準依存管理として扱う記述
- root `requirements.txt` を標準依存管理として扱う記述
- 旧 root `libkoiki/` を正本として扱う記述
- root `app/` を実装正本として扱う記述
- root `alembic/` を migration 正本として扱う記述
- `app.main:app` を新規推奨 ASGI path として扱う記述
- 生成 cache / tool-local state を Git 保管対象として扱うこと

## 分類ルール

### A. 現行運用文書

README、setup、Docker、security audit、test guide、SAML guide、agent guidance など、開発者や agent が現在参照し得る文書。

原則として現行構成へ更新する。

### B. 履歴・計画文書

release note、過去 design、v0.7 task instruction、backend audit work plan など、過去判断や作業経緯を記録する文書。

原則として内容を書き換えず、必要に応じて archive / historical reference として扱う。

### C. agent / tool 由来ファイル

`.serena/`、`.kiro/`、`.specstory/`、`.claude/`、`.github/instructions/` など。

現在の agent 作業に影響するものは更新する。生成 cache や会話履歴は Git 管理の要否を判断する。

## Docs Cleanup FU-1: tracked agent/tool history の整理

対象:

- `.serena/cache/**`
- `.serena/memories/*.md`
- `.serena/project.yml`
- `.kiro/steering/*.md`
- `.specstory/history/**`
- `.specstory/cli/**`

目的:

- tool-generated cache と共有すべき知識を分離する
- 現行構成と矛盾する agent/tool history が作業標準として読まれないようにする
- Git 管理対象に残すものとローカル生成物に戻すものを明確にする

推奨方針:

- `.serena/cache/**`
  - 再生成可能な cache として Git 管理から外す
  - `.serena/.gitignore` の `/cache` は維持する
- `.serena/project.yml`
  - Serena project 設定として Git 管理を継続してよい
- `.serena/memories/*.md`
  - 残す場合は現行 `uv` / `components` 構成へ更新する
  - 共有不要なら削除候補とする
- `.kiro/steering/*.md`
  - 現在も使うなら現行構成へ更新する
  - 使わないなら削除または archive 候補とする
- `.specstory/history/**`
  - 会話履歴として残す価値はあるが、現行 docs としては危険
  - 原則として Git 管理から外すか、archive / historical 扱いを明確にする
- `.specstory/cli/**`
  - tool 設定として必要か確認する
  - project-local 設定として共有する価値がなければ Git 管理対象から外す

完了条件:

- tracked 生成 cache の扱いが決まっている
- `.serena/memories` を残す場合、Poetry / 旧 path 前提が現行構成へ更新されている
- `.kiro/steering` を残す場合、Poetry / 旧 path 前提が現行構成へ更新されている
- `.specstory/history` を残す場合、現行 docs ではなく履歴であることが明確になっている
- Git 管理対象に残す agent/tool 由来ファイルを説明できる

実施結果:

- `.serena/cache/python/document_symbols_cache_v23-06-25.pkl` を Git 管理対象から外した
- `.specstory/history/**` を Git 管理対象から外し、`.specstory/.gitignore` で `/history` を無視するようにした
- `.serena/memories/*.md` の主要な構成・コマンド・完了チェックリストを `uv` / `components` 構成へ更新した
- `.kiro/steering/*.md` の Poetry コマンド、旧 root `libkoiki/` / `app/` 正本表現を現行構成へ更新した

検証結果:

- `git ls-files .specstory/history .serena/cache`
  - 出力なし
- `rg -n -i "poetry|poetry\.lock|requirements\.txt" .serena .kiro --glob "*.md"`
  - `.serena/memories/project_structure.md` の履歴資料注記のみ

状態:

- `完了`

検証コマンド:

```powershell
git ls-files .serena .kiro .specstory
rg -n -i "poetry|poetry\.lock|requirements\.txt|app\.main:app|libkoiki/|alembic/" .serena .kiro .specstory --glob "*.md"
git status --short
```

## Docs Cleanup FU-3: 現行 runbook / guide の軽微な齟齬修正

対象候補:

- `ops/README.md`
- `DOCKER_SETUP.md`
- `docs/archive/README-deploy.md`
- `docs/archive/ENTERPRISE_DEPENDENCY_STRATEGY.md`
- その他 root / docs 配下の現行運用 guide

目的:

- 現行運用 guide に残る小さな齟齬を解消する
- 履歴資料と現行 runbook の境界をさらに明確にする

既知候補:

- `ops/README.md`
  - 旧 `ops/requirements.txt` は Dependabot のスキャン補助として手動管理していたが、`uv.lock` / Dependabot `uv` ecosystem 前提へ移行したため削除する
- `DOCKER_SETUP.md`
  - 旧 Docker / Compose ファイルは「削除予定」ではなく、現時点では削除済みとして扱う
- `docs/archive/README-deploy.md`
  - 現行 deploy guide として維持するか、履歴注記付き資料として残すかを判断する
- `docs/archive/ENTERPRISE_DEPENDENCY_STRATEGY.md`
  - historical reference として残すか、archive 側へ寄せるかを判断する

完了条件:

- 現行 runbook が古いファイルや旧依存方式を標準として説明していない
- 履歴資料として残す文書にはその旨が明記されている
- 軽微な齟齬が issue / follow-up なしで後続作業を妨げない

実施結果:

- `ops/README.md` から `ops/requirements.txt` の記載を削除した
- `ops/requirements.txt` を削除し、Dependabot は root `uv.lock` を対象にする方針へ寄せた
- `DOCKER_SETUP.md` の旧 Docker / Compose ファイル説明を「削除予定」から「廃止済み」へ更新した
- `docs/archive/README-deploy.md` を履歴資料として明記し、現行 Docker 運用の参照先を `DOCKER_SETUP.md` / `start-docker.ps1` にした
- `docs/dev/local_setup.md` と `docs/dev/setup.md` の起動例を `koiki_ref_app.asgi:app` 標準へ更新した

検証結果:

- 現行 agent guidance / security audit / Docker setup には Poetry 標準手順が残っていない
- 残る `Poetry` 記述は履歴資料、過去計画、または本 follow-up 計画の文脈に分類できる

状態:

- `完了`

検証コマンド:

```powershell
rg -n -i "poetry|requirements\.txt|削除予定|app\.main:app|libkoiki/|alembic/" README.md docs/archive/README-deploy.md DOCKER_SETUP.md docs/security/SECURITY_AUDIT_COMMANDS.md ops/README.md docs
```
