# KOIKI-FW v0.7 作業タスク分解

## 目的

本書は、[v0.7-directory-reorganization-plan.ja.md](./v0.7-directory-reorganization-plan.ja.md) を、実作業可能なタスク単位へ分解したものです。

設計方針は次の通りです。

- タスクは細かく分けるが、単なる機械的分割ではなく意味単位で切る
- すべてのタスクに検証フェーズを持たせる
- 前工程の結果を確認してから次工程へ進む
- 大規模移動より先に、責務整理、依存整理、起動整理を行う

## 運用ルール

- 各タスクは、完了時に「変更内容」「未解決事項」「検証結果」を記録する
- 次タスクへ進む条件は、当該タスクの検証が通っていること
- タスクの途中で前提が変わった場合は、後続タスクへ進まず、計画書と本タスクリストを更新する
- タスク手順書を変更した場合は、必要に応じて計画書とタスク分解書も同時に見直し、3 文書の前提を同期させる
- 計画書を更新する場合は、日本語版と英語版の `v0.7-directory-reorganization-plan` を同期対象として扱う
- 破壊的な移動や rename は、互換導線を確保してから行う

## ステータス記法

- `未着手`
- `進行中`
- `完了`
- `保留`

## タスク完了条件

各タスクは、少なくとも次を満たした時点で `完了` とします。

- 当該タスクの目的に対して実装または判断結果が記録されている
- 本文中の「検証」が実施され、結果が確認できる
- 後続タスクへ渡す未解決事項がある場合、それが明示されている
- 実コード変更を伴う場合、対応する doc / script / test の追随要否が確認されている

## タスク実行記録テンプレート

各タスクの実行後は、PR、Issue、作業メモなどに次の形式で残すことを推奨します。

```text
Task:
- Task X-Y: <タスク名>

変更内容:
- <実施した変更>

未解決事項:
- <残課題。なければ「なし」>

検証結果:
- <実施した確認手順>
- <結果>

次タスクへ渡す事項:
- <後続タスクへの申し送り。なければ「なし」>
```
